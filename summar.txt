# Phoenix Studio: Master Architecture & Implementation Plan Summary

## 1. Tech Stack Overview
- **Frontend**: React (Vite) + Vanilla CSS + Bootstrap
- **Diagram Engine**: React Flow (node-based editor)
- **Backend**: FastAPI (Python), Jinja2 for code templating
- **Packaging**: ZIP file with `main.py`, `.env`, and `requirements.txt`

## 2. Node-to-SDK Mapping
| **Visual Node**       | **Phoenix SDK Equivalent**                                                                 |
|-----------------------|------------------------------------------------------------------------------------------|
| Agent                 | `from phoenix.framework.agent import Agent`                                               |
| LLM (OpenAI/Local)    | `from phoenix.services.llm.openai import OpenAILLM`                                      |
| Memory (Hybrid)       | `from phoenix.framework.agent.memory.hybrid import HybridMemory`                         |
| Tool (Default/Custom) | `@tool` decorator for custom tools or `ToolRegistry.load_default()`                      |

## 3. Code Generation Flow
1. **Frontend**: Serializes React Flow canvas to JSON (nodes/edges).
2. **Backend**: FastAPI endpoint `/api/generate` parses the graph.
3. **Template**: Uses Jinja2 to generate Python script (e.g., async `main()` function).
4. **Output**: Packages into ZIP with:
   - Generated `main.py`
   - `.env` for API keys
   - `requirements.txt` (`phx-ashborn[full]`).

## 4. Phase Roadmap
- **Phase 1**: Setup repo, initialize FastAPI/React Vite.
- **Phase 2**: Build React Flow Canvas + Property Sidebar.
- **Phase 3**: Implement backend generation API.
- **Phase 4**: Polish UI, validate project execution.
